Detox locks API

Usage

This API is used to protect / unprotect datasets and blocks from dynamo deletions. Locks are identified by the protected item (dataset or block), site, group, requesting user, and optionally the "service name". Service name is used if a user is managing locks in behalf of some system (e.g. Unified). Only users and services with permissions can make changes to the lock database (read-only functions are available to everyone). Please write a message to the HN hn-cms-dmDevelopment (DataManagement Developers) to obtain a permission.

There is NO browser-friendly interface for detox locks at the moment, aside from this manual page. Use cURL to access the API: 

        curl --cert proxy --key proxy 'https://dynamo.mit.edu/registry/detoxlock/command[?options]'
where proxy is a valid CMS grid proxy and command is one of the commands listed below.
Note that authentication with grid proxies work only with cURL+OpenSSL, which is not the default build in Scientific Linux and CentOS. CMS environment sourced from CVMFS provides a working configuration.

Examples

  • Lock a dataset /A/B/C: 
              detoxlock/lock?item=/A/B/C&expires=2017-12-31&comment=Lock%20for%20production
    When the lock is successful, a "lock ID" will be returned. This ID can be used to release or re-lock the same item.
  • Lock a dataset /A/B/C at site X: 
              detoxlock/lock?item=/A/B/C&sites=X&expires=2017-12-31&comment=Lock%20for%20production
  • Lock a dataset /A/B/C as a service user "prodsystem": 
              detoxlock/lock?service=prodsystem&item=/A/B/C&expires=2017-12-31&comment=Lock%20for%20production
    When the lock is successful, a "lock ID" will be returned. This ID can be used to release or re-lock the same item.
  • Release a lock with ID 20: 
              detoxlock/unlock?lockid=20
    Only locks created with your grid user name can be released.
  • Release a lock with item and site names: 
              detoxlock/unlock?item=/A/B/C&sites=X
  • Release a lock for the service "prodsystem": 
              detoxlock/unlock?service=prodsystem&item=/A/B/C
  • List all locks created before 2017-04-01: 
              detoxlock/list?created_before=2017-04-01&user=all

Commands

  • lock

    Creates or update a lock and enable it. Takes options service, format, return, lockid, item, sites, groups, expires, and comment. At least lockid or item must be specified.

  • unlock

    Updates a lock and disable it. Takes options service, format, return, lockid, item, sites, groups, expires, and comment. At least lockid or item must be specified. All locks matching the condition are disabled.

  • set

    Upload a JSON or an XML as POST data with this command. Disables all locks owned by the user first and loads the content of the uploaded data. The JSON format is 

                [{"item": "item_name", "sites": "sites_pattern", "groups": "groups_pattern", "expires": "2017-12-31 00:00:00", "comment": "comment_string"}, ...]
    The XML format is 
                <locks>
              <lock>
                <item>item_name</item>
                <sites>sites_pattern</sites>
                <groups>groups_pattern</groups>
                <expires>2017-12-31 00:00:00</expires>
                <comment>comment_string</comment>
              </lock>
              ...
            </locks>

    Sites, groups, and comment are optional. Takes options service, format and return. format sets the format of both the uploaded and returned data.
  • list

    Returns a list of locks. Locks can be enabled or disabled. By default, return only locks created by the user's grid user name. List can be narrowed by options. Takes options service, format, return, lockid, item, sites, groups, user, created_before, created_after, expires_before, and expires_after.

Options

  • service

    Name of the CMS service the user is managing the locks for. When specified, authorization is checked for the user-service combination.

    Values
    A service name string.

  • format

    In all except the set command, specify the format of the text output of the API. When used with set, also specifies the uploaded data format.

    Values
    json (default), xml

  • return

    Specifies whether the API outputs a full detail of updated lock information. If set to no, only the summary is given.

    Values
    yes (default), no

  • lockid

    A unique identifier of locks. Displayed in the text output after all commands when return=yes. lockid can be used to update the state of the locks.

    Value
    An integer.

  • item

    A dataset or block name. Wildcard (*) is allowed and will match strings of any length.

    Value
    A string.

  • sites

    Site name pattern. Wildcard (*) is allowed and will match strings of any length.

    Value
    A string.

  • groups

    PhEDEx group name pattern. Wildcard (*) is allowed and will match strings of any length.

    Value
    A string.

  • expires

    Expiration date of the lock. Expiration is used as a guidance but is not enforced.

    Value
    A string of format YYYY-mm-dd[ HH:MM:SS]. Note that the white space must be replaced by %20 when passed as a GET string from the command line.

  • comment

    Lock comment.

    Value
    A free-format string. Note that white spaces must be replaced by %20 (and all other non-alphanumeric characters with their URL-encoded values) when passed as a GET string from the command line.

  • user

    Used only with list command. When set to all, lists locks by all users.

    Value
    self (default), all

  • created_before

    Used only with list command. Limits the list to locks created before the given timestamp.

    Value
    A string of format YYYY-mm-dd[ HH:MM:SS]. Note that the white space must be replaced by %20 when passed as a GET string from the command line.

  • created_after

    Used only with list command. Limits the list to locks created after the given timestamp.

    Value
    A string of format YYYY-mm-dd[ HH:MM:SS]. Note that the white space must be replaced by %20 when passed as a GET string from the command line.

  • expires_before

    Used only with list command. Limits the list to locks expiring before the given timestamp.

    Value
    A string of format YYYY-mm-dd[ HH:MM:SS]. Note that the white space must be replaced by %20 when passed as a GET string from the command line.

  • expires_after

    Used only with list command. Limits the list to locks expiring after the given timestamp.

    Value
    A string of format YYYY-mm-dd[ HH:MM:SS]. Note that the white space must be replaced by %20 when passed as a GET string from the command line.
